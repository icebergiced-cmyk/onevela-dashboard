# ระบบตรวจสอบ (Audit) และการแก้ไขข้อมูล

> ตอบคำถาม: "ฉันจะรู้ได้อย่างไรว่าระบบบันทึกครบ" + "duplicate ตรวจอย่างไร" + "แก้สถานะ/ราคา ทำที่ไหน"

---

## 1️⃣ Audit Report — ตรวจว่าระบบบันทึกครบ

ทุกครั้งที่ convert Excel จะสร้าง Excel report ในโฟลเดอร์ `_audit_reports/audit_YYYYMMDD_HHMM.xlsx` ที่มี **5 sheet** ให้เทียบ:

### Sheet 1: สรุปการเทียบ
- จำนวน row ทั้งหมดใน Excel
- ยอดรวมจาก row "ทั้งหมด" ของ Excel
- ยอดรวมที่ Dashboard บันทึก
- **แถบเขียว ✅ = ยอดตรง** (บันทึกครบทุก row ของแปลง)
- แถบเหลือง ⚠️ = ยอดต่าง แต่เท่ากับ overhead ที่ข้าม (ปกติ)
- แถบแดง ❌ = ยอดผิด ต้องตรวจสอบ

### Sheet 2: รายการที่ไม่ได้บันทึก
ทุก row ที่ระบบข้ามไป พร้อมเหตุผล เช่น:
- `ไม่ใช่รายการของแปลง (overhead)` — แคมป์คนงาน, ค่าใช้จ่ายสิ้นเปลือง
- `ไม่พบแปลงในระบบ` — แปลงที่ไม่อยู่ใน 483 แปลงดั้งเดิม
- `อ่านวันที่ไม่ได้` — format วันที่ผิด

ดูได้ว่าเงินที่หายไปจาก Dashboard ไปอยู่ตรงไหน

### Sheet 3: รายการซ้ำ
รายการที่มี (แปลง + วันที่ + สินค้า + จำนวน + ยอด) เหมือนกันเป๊ะ
- ระบบจะ **ไม่ลบให้** แค่แจ้งเตือน เพราะบางครั้งซื้อจริง 2 ครั้งในวันเดียวกัน
- คุณตัดสินใจเองว่าเป็นซ้ำพลาดหรือซ้ำจริง
- ถ้าซ้ำพลาด: ลบใน Excel ก่อน export ใหม่ แล้ว run อีกครั้ง

### Sheet 4: เทียบยอดทุกแปลง ⭐ จุดสำคัญ
ทุกแปลงที่มี transaction:

| แปลง | ไทป์ | Excel ยอดรวม | Dashboard ยอดรวม | ส่วนต่าง | สถานะ |
|------|------|---------------|-------------------|----------|--------|
| 1    | A    | 5,856.29     | 5,856.29         | 0        | ✓ ตรง  |
| 2    | A    | 374,199.68   | 374,199.68       | 0        | ✓ ตรง  |
| ...  | ...  | ...          | ...              | ...      | ...   |

ถ้ามีแปลงไหน ❌ ส่วนต่าง ≠ 0 = แดง แจ้งว่าตัวเลขผิด ผมจะดูให้

### Sheet 5: สินค้าใหม่ที่ต้องตรวจ category
รายการสินค้าที่ Excel มี แต่ระบบยังไม่รู้ว่าจัดหมวดไหน
- มี column "Category ที่ระบบเดา" + "ถูกต้องไหม?" + "Category ที่ถูก"
- คุณกรอกตอบกลับมา ผมเพิ่มเข้า dictionary ครั้งหน้าจะ classify ถูก

## 2️⃣ Duplicate Detection — ตรวจรายการซ้ำ

### นิยามรายการซ้ำของระบบ

ระบบถือว่าซ้ำเมื่อ **5 ฟิลด์เหมือนกันทุกตัว**:
- แปลง (column B)
- วันที่ (column A)
- ชื่อสินค้า (column C)
- จำนวน (column D)
- ยอดรวม (column G)

### พฤติกรรมระบบ

- **ไม่ลบให้อัตโนมัติ** — อาจเป็นการซื้อจริงในวันเดียวกัน
- **แจ้งเตือนใน Sheet 3** พร้อมระบุ Excel row number ที่ซ้ำกัน
- บอก "ส่วนเกิน" ถ้าคิดว่าเป็นซ้ำพลาด เพื่อให้คุณเห็นผลกระทบทางตัวเลข

### วิธีจัดการเมื่อเจอซ้ำ

1. เปิด Excel ดู row ที่ระบบแจ้ง
2. ตัดสินใจว่าเป็นซ้ำพลาดหรือซ้ำจริง
3. ถ้าซ้ำพลาด: ลบใน Excel แล้ว export ใหม่
4. ส่งให้ Claude อีกครั้ง

> 💡 **เคล็ดลับ:** ปกติแล้ว Ecount จะไม่ duplicate ถ้า import ถูก ถ้าเจอซ้ำเยอะๆ ลองตรวจที่ Ecount ก่อน — อาจมี user import ซ้ำ

## 3️⃣ แก้ไขข้อมูลใน Dashboard — ทำอย่างไร

### สรุปสั้น: ไม่สามารถแก้ใน Dashboard โดยตรงได้ — เพราะเป็น HTML read-only

แต่มี **3 ทางแก้** ขึ้นกับว่าจะแก้อะไร:

### A) ข้อมูลต้นทุน (รายการเบิก, ค่าวัสดุ, ค่าแรง)
**→ แก้ที่ Ecount ERP แล้ว export Excel ใหม่ ส่งให้ Claude**

ระบบ pipeline เป็น Excel → Dashboard ดังนั้น "ต้นทาง" คือ Ecount เสมอ

### B) ข้อมูล metadata (สถานะขาย, ราคาขาย, บ้านเลขที่, หมายเหตุ)
**→ บอก Claude โดยตรง**

ตัวอย่างประโยคที่คุณพิมพ์ใน Cowork chat:
- "แปลง 5 ทำสัญญาแล้ว เปลี่ยน status เป็น สัญญา"
- "แปลง 12 ลูกค้าโอนเรียบร้อย เปลี่ยนเป็น โอนแล้ว ราคาสุทธิ ฿4,100,000"
- "แปลง 7 บ้านเลขที่ 89/7 ใหม่ (จาก 89/6)"
- "แปลง 23 หมายเหตุ: ลูกค้าขอแก้ floorplan ห้องครัว"

Claude จะ:
1. รัน `update_metadata.py` แก้ `plots_metadata.json`
2. รัน `excel_to_dashboard.py` rebuild HTML
3. Deploy ขึ้น Netlify
4. ส่ง confirm

### C) แก้พร้อมกันหลายแปลง (เช่น โอนพร้อมกัน 10 หลัง)
**→ ส่ง Excel มี column: plot, status, priceNet, houseNo, note**

ตัวอย่าง:

| plot | status   | priceNet | note               |
|------|----------|----------|---------------------|
| 12   | โอนแล้ว  | 4100000  | โอนเมื่อ 15 พ.ค.     |
| 13   | โอนแล้ว  | 4050000  |                     |
| 14   | สัญญา    |          |                     |

ส่งให้ Claude พร้อมคำว่า "batch update metadata" — Claude จะ update ทั้งหมดในครั้งเดียว

## ทำไมไม่ทำให้ Dashboard แก้ได้ในตัว?

ทำได้แต่ต้องเลือกระหว่าง 2 รูปแบบ:

### ตัวเลือก 1: Phase 2 (Google Sheet — แนะนำถ้าทีมจะใช้แก้ข้อมูลบ่อย)
- ย้าย metadata ไป Google Sheet
- ทีมเข้า Sheet แก้ได้พร้อมกัน
- Dashboard ดึงจาก Sheet สดๆ ทุกครั้งที่เปิด
- ลงทุน 1-2 ชม. ตั้งระบบครั้งแรก
- เปิดให้แก้ได้บนมือถือเลย

### ตัวเลือก 2: Web app จริง (Retool / Supabase)
- มีหน้า login + ฟอร์มแก้ในเว็บ
- ใช้เวลาทำ 2-3 วัน
- ดูเป็นมืออาชีพมาก
- เหมาะถ้าจะ scale up

### ตัวเลือก 3: Status quo — แจ้ง Claude
- ฟรี ใช้ได้เลย
- เหมาะกับการแก้ที่ไม่บ่อย (1-2 ครั้ง/สัปดาห์)
- ต้องผ่าน Claude ทุกครั้ง

ตอนนี้ใช้ตัวเลือก 3 ไปก่อน ถ้ารู้สึกเหนื่อย หรือทีมอยากแก้เอง บอก Claude — ผมจะตั้ง Phase 2 ให้
