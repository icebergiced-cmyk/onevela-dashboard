# คู่มืออัพเดตข้อมูล Dashboard รายวัน

> สำหรับ One Vela cost dashboard — `https://onevela-cost.netlify.app`

---

## ระบบทำงานอย่างไร

```
[Excel จาก Ecount ERP] → [Python converter] → [index.html ใหม่] → [Netlify] → [ทีมเห็น]
       ↑ คุณ                  ↑ Claude/script         ↑ auto              ↑ auto         ↑ refresh มือถือ
```

ขั้นตอนสั้นๆ คือ:
1. คุณ export Excel จาก Ecount
2. ส่งให้ Claude (หรือรัน script เอง)
3. รอ 1-2 นาที — Dashboard อัพเดทบนมือถือทีมทันที

---

## วิธีที่ 1 — ส่ง Excel ให้ Claude (ง่ายสุด)

### ทุกวันทำแบบนี้:

1. **Export Excel จาก Ecount**
   - เมนู: ขาย/รายงาน → "สถานะการขาย" (หรือชื่อรายงานที่ใช้ export ตัวอย่างที่ส่งให้)
   - เลือกช่วงเวลา: ตั้งแต่ 1 ม.ค. 2569 ถึง วันปัจจุบัน
   - Export เป็น Excel (.xlsx)

2. **เปิดแชท Cowork กับ Claude**
   - ลากไฟล์ Excel ใส่แชท
   - พิมพ์ว่า: **"อัพเดต Dashboard ด้วย Excel นี้"**

3. **Claude จะ:**
   - รัน converter → สร้าง index.html ใหม่
   - Deploy ขึ้น Netlify
   - ส่งลิงก์ confirm พร้อมสรุปข้อมูล (กี่แปลง active, ต้นทุนรวมเท่าไหร่)
   - แจ้งถ้ามีรายการใหม่ที่ระบบยังไม่รู้จัก category

4. **ทีมเห็นข้อมูลใหม่ภายใน 5 นาที**
   - ถ้าใครเปิด Dashboard ค้างไว้ ต้อง refresh 1 ครั้ง
   - บนมือถือ: เปิด-ปิดแอป Dashboard ใหม่
   - URL: https://onevela-cost.netlify.app

## วิธีที่ 2 — รัน Script เอง (เหนื่อยน้อยลง ไม่ต้องคุยกับ Claude)

หากต้องการอัพเดทเองโดยไม่ผ่าน Claude:

### ครั้งแรก — ติดตั้ง Python (5 นาที, ทำครั้งเดียว)

1. เปิด Terminal บน Mac
2. รันคำสั่งติดตั้ง:
   ```bash
   pip3 install openpyxl
   ```

### ทุกวันทำแบบนี้ (30 วินาที)

1. Export Excel จาก Ecount ตามปกติ
2. เปิด Terminal
3. รัน:
   ```bash
   cd "/Users/imacm4/Desktop/one vela main file/dashboard-online/_tools"
   python3 excel_to_dashboard.py "/path/to/your/excel.xlsx"
   ```
4. จะได้ไฟล์ `dashboard-deploy.zip` ใหม่ในโฟลเดอร์ `dashboard-online`
5. เปิด https://app.netlify.com/projects/onevela-cost/deploys
6. ลาก `dashboard-deploy.zip` ลงในกล่อง drag-drop
7. รอ 30 วิ — เสร็จ

## รูปแบบ Excel ที่ระบบรองรับ

ระบบรองรับ Excel export จาก Ecount ERP ที่มีรูปแบบนี้:

| Column | ชื่อ | ตัวอย่าง |
|--------|------|---------|
| A | ประจำวัน | `06 เม.ย. 2569` |
| B | ลูกค้า/ผู้ขาย | `บ้านเดี่ยว 2 ชั้น แปลง 3` |
| C | ตามสินค้า | `ปูนฉาบอิฐมวลเบา เสือมอร์ตาร์ 50กก.` |
| D | จำนวน | `10` |
| E | จำนวนเงิน | `940` |
| F | ภาษี | `(ว่างได้)` |
| G | ยอดรวม | `940` |

**Plot types ที่รองรับ** (ใน column B):
- `บ้านเดี่ยว 2 ชั้น แปลง [N]` → A ULTRA
- `บ้านเดี่ยวชั้นเดียว แปลง [N]` → B MINUTES
- `บ้านแฝดชั้นเดียว แปลง [N]` → C SECOND
- `บ้านทาวน์โฮม แปลง [N]` → D MOMENT (หรือ `ทาวน์โฮม แปลง [N]`)

**Row ที่ระบบจะข้าม:**
- Header rows
- Summary rows ที่ลงท้ายด้วย "ทั้งหมด"
- รายการที่ไม่ผูกกับแปลง (เช่น `แคมป์คนงาน`, `ค่าใช้จ่ายสิ้นเปลือง`)

## สิ่งที่ต้องสังเกต

### 1. รายการใหม่ที่ระบบยังไม่รู้จัก category

ทุกครั้งที่ converter เจอชื่อสินค้าใหม่ที่ไม่มีในระบบ จะมี fallback:
- ถ้าขึ้นต้นด้วย `ช่าง` หรือมีคำว่า `ค่าบริการ` → `ค่าแรง`
- อย่างอื่น → `วัสดุสิ้นเปลือง/อื่นๆ`

**ถ้ามีหมวดผิด แจ้ง Claude เพื่อปรับ category rules** เช่น:
> "ช่วยจัด 'ปูนฉาบสูตรพิเศษ XYZ' ให้อยู่หมวด 'งานก่อ-ฉาบ ผนัง' หน่อย"

Claude จะเพิ่มเข้า `item_categories.json` ครั้งต่อไปจะ classify ถูก

### 2. รายการที่ไม่ผูกกับแปลง

รายการประเภท "แคมป์คนงาน", "ค่าใช้จ่ายสิ้นเปลือง" หรือ overhead ทั่วไป จะไม่ขึ้นบน Dashboard เพราะแยกตามแปลงไม่ได้ ตอนนี้เก็บไว้ที่ ERP เป็นข้อมูลภาพรวม

ถ้าอยากให้ Dashboard แสดงตรงนี้ด้วย ต้องเพิ่มหน้า "ค่าใช้จ่ายรวมโครงการ" — แจ้ง Claude ได้

### 3. ข้อมูล metadata 483 แปลง (ราคา, สถานะ, เลขโฉนด)

ข้อมูลส่วนนี้ไม่ได้มาจาก Excel — มาจากข้อมูล setup ดั้งเดิมที่เก็บใน `plots_metadata.json`

ถ้าข้อมูลแปลง (เช่น สถานะการขาย "ว่าง" → "จอง" → "โอนแล้ว", หรือบ้านเลขที่) **เปลี่ยน** ต้อง update `plots_metadata.json` แยกต่างหาก แจ้ง Claude ได้

## ปัญหาที่อาจเจอ

**ทีมยังเห็นข้อมูลเก่าบนมือถือ**

→ Service Worker cache ทำงาน ปกติ refresh 1 ครั้งหาย ถ้ายังไม่หาย: เปิด-ปิดแอป Dashboard บนมือถือใหม่ หรือ Settings → Clear browser cache

**Excel แสดงต้นทุนต่างจาก Dashboard**

→ ตรวจ summary rows ที่ "ทั้งหมด" — Dashboard จะคำนวณจาก detail rows เอง อาจเจอเรื่อง:
- รายการที่ไม่ผูกแปลง (เช่น แคมป์คนงาน) ไม่ขึ้น Dashboard
- รายการบางตัว category ผิด ทำให้ดูเหมือนหายไป (จริงๆ ไปอยู่หมวดอื่น)

**Deploy zip ขึ้น Netlify แล้ว URL ไม่อัพเดท**

→ รอ 30-60 วินาที แล้ว force refresh (Cmd+Shift+R) หรือเปิด incognito ทดสอบ
