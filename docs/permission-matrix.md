# Permission Matrix — ระบบ One Vela

> เอกสาร Round 1 · กำหนดสิทธิ์ 4 role ในหน้า plot-detail และเครื่องมือต่างๆ

## 4 Roles

| role | ใคร | สรุปสิทธิ์ |
|---|---|---|
| `admin` | พี่ Ice | เห็นทุกอย่าง · แก้ได้ทุกอย่าง · อนุมัติ · จัดการผู้ใช้ |
| `foreman` | โฟร์แมน | แก้เฉพาะงานก่อสร้าง+รูป+ปัญหา · ส่วนอื่นดูไม่ได้ |
| `sales` | ทีมขาย | แก้งานขาย · ดูงานก่อสร้างได้ (อ่านอย่างเดียว) |
| `finance` | การเงิน | แก้การเงิน · ดูส่วนอื่นได้ (อ่านอย่างเดียว) |

## Matrix — สิทธิ์ต่อ section

ค่า: **แก้** = ดู+แก้ได้ · **ดู** = ดูอย่างเดียว · **✕** = ไม่เห็น section เลย

| Section | admin | foreman | sales | finance |
|---|:---:|:---:|:---:|:---:|
| ข้อมูลแปลง (หัวบ้าน/ราคา/สถานะ) | แก้ | ดู | แก้ | ดู |
| งานขาย — ใบเสนอราคา/จอง/สัญญา/ใบเสร็จ | แก้ | ✕ | แก้ | ดู |
| การเงิน — ผ่อนดาวน์/ชำระเงิน | แก้ | ✕ | ดู | แก้ |
| งานก่อสร้าง — milestone/ความคืบหน้า | แก้ | แก้ | ดู | ดู |
| รูปหน้างาน | แก้ | แก้ (อัป) | ดู | ดู |
| ปัญหาหน้างาน (issues) | แก้ | แก้ | ดู | ดู |
| อนุมัติอัปเดตงานก่อสร้าง | แก้ | ✕ | ✕ | ✕ |
| จัดการผู้ใช้ (tab `users`) | แก้ | ✕ | ✕ | ✕ |

## กฎการแสดงผล

- เปิดหน้า → ตรวจ JWT → รู้ role → ซ่อน/แสดง section ตามตาราง
- section ที่เป็น **✕** = ไม่ render เลย (ไม่ใช่แค่ disable)
- section ที่เป็น **ดู** = render แต่ปุ่มแก้/บันทึก/อัปโหลด ถูกซ่อน
- ปุ่ม "อัปเดตงาน" (งานก่อสร้าง) = เห็นเฉพาะ `admin` + `foreman`
- ปุ่ม "อนุมัติ/ปฏิเสธ" = เห็นเฉพาะ `admin`

## Flow การอนุมัติงานก่อสร้าง

```
foreman ส่งอัปเดต → construction_updates.status = pending
                         │
                  admin ตรวจ
                    │         │
                อนุมัติ      ปฏิเสธ
                    │         │
      status=approved    status=rejected
      + อัปเดต            + rejection_reason
        plot_construction   (โฟร์แมนแก้แล้วส่งใหม่)
```

- เฉพาะ `admin` เปลี่ยน `status` ของ construction_updates ได้
- `foreman` สร้างได้อย่างเดียว (status เริ่มต้น = `pending`) แก้ของที่ approved แล้วไม่ได้
- เมื่ออนุมัติ — `approved_by` = user_id ของ admin, `approved_at` = เวลาปัจจุบัน

## หมายเหตุความปลอดภัย

- Permission ตรวจ **2 ชั้น**: ฝั่งหน้าเว็บ (ซ่อน UI) **และ** ฝั่ง Apps Script (ตรวจ JWT+role ก่อนเขียนข้อมูลทุกครั้ง)
- ห้ามเชื่อ role จากฝั่ง client อย่างเดียว — Apps Script ต้องตรวจซ้ำเสมอ (กันการปลอม request)
- ผู้ใช้ที่ `active = 0` ใน tab `users` → ปฏิเสธทุก request
