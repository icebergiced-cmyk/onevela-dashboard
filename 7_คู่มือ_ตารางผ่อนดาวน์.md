- **type:** จอง / สัญญา / ดาวน์ / กู้ / โอน / อื่นๆ

## ตัวอย่างข้อมูลที่ผมเตรียมไว้

ในไฟล์ template ผม pre-fill **5 แปลงที่มีสถานะขายแล้ว × 15 งวด/แปลง = 75 แถว** ให้ดูเป็นตัวอย่าง

ลำดับงวดมาตรฐาน:
```
1. เงินจอง         ~฿10,000      ประเภท: จอง
2. เงินทำสัญญา      ~฿50,000      ประเภท: สัญญา
3-14. ผ่อนดาวน์ 12 งวด  งวดละ ~฿20-50K  ประเภท: ดาวน์
15. ส่วนกู้+ค่าโอน  ราคาที่เหลือ   ประเภท: กู้
```

ปรับให้เหมาะกับเงื่อนไขจริงของแต่ละแปลงได้ตามต้องการ

## ⚙️ วิธี Setup (10 นาที)

### ขั้นที่ 1 — Import xlsx ใหม่เข้า Sheet เดิม

1. เปิด Google Sheet เดิม
2. **File → Import** เลือก `houses_metadata.xlsx` (ตัวล่าสุด มี 3 tabs แล้ว)
3. เลือก **"Replace spreadsheet"** กด Import
4. ตอนนี้ Sheet มี 3 tabs: คู่มือ + metadata + installments

[📂 เปิดไฟล์ xlsx ใหม่](computer:///Users/imacm4/Desktop/one vela main file/dashboard-online/_tools/houses_metadata.xlsx)

### ขั้นที่ 2 — Publish tab "installments" เป็น CSV

1. คลิกที่ tab **"installments"** ที่ด้านล่าง Sheet (ต้องอยู่ที่ tab นี้ก่อน)
2. **File → Share → Publish to web**
3. ใน popup เลือก:
   - **Link** tab
   - Dropdown ซ้าย: **"installments"** (สำคัญ — ไม่ใช่ metadata)
   - Dropdown ขวา: **"Comma-separated values (.csv)"**
   - ✓ Automatically republish when changes are made
4. กด **Publish**
5. Copy URL ที่ได้ — จะเป็น URL ที่ขึ้นต้น `https://docs.google.com/spreadsheets/d/e/.../pub?gid=XXX&single=true&output=csv`

### ขั้นที่ 3 — ส่ง URL ที่ 2 ให้ผม

ส่งใน chat: "URL installments: https://..."

ผมจะ:
- เพิ่มลง `sheet_config.json`
- Re-deploy Dashboard
- ทดสอบให้ดูว่า merge ถูก

### ขั้นที่ 4 — Republish CSV ของ metadata ใหม่ (ถ้า URL เปลี่ยน)

ถ้า import xlsx ทำให้ Sheet ID เปลี่ยน (บางทีก็เปลี่ยน) — Publish tab metadata ใหม่ด้วย แล้วส่ง URL ที่อัพเดทให้ผม

## 💡 Tips การกรอกข้อมูล

**1. กรอกแถวเดียวก่อน — ทดสอบ flow**

แทนที่จะกรอก 75 แถวทันที ลองกรอกแค่ 1 แปลง × 3 งวด ดูว่า Dashboard แสดงถูกก่อน

**2. ใช้ Google Sheets "Drag handle" เพื่อ duplicate งวด**

- เลือก row งวด 1 ของแปลง 5
- ลากที่มุมขวาล่างเซลล์ลง 12 แถว
- จะ duplicate งวด 1-12 ทันที
- แค่แก้ seq + dueDate + dueAmount ของแต่ละแถว

**3. สถานะ "จ่ายบางส่วน"**

ใช้เมื่อลูกค้าจ่ายไม่เต็มจำนวน เช่น dueAmount = 50,000 แต่ paidAmount = 30,000 → ใส่ status = "จ่ายบางส่วน"

**4. ยกเลิก vs ลบ**

ห้ามลบแถว — เปลี่ยน status เป็น "ยกเลิก" แทน เพื่อเก็บ history

**5. กรณีพิเศษ — เปลี่ยนแผนผ่อน**

ถ้าลูกค้าขอเลื่อนวันจ่าย: อย่าลบงวดเก่า ให้:
- งวดเก่า: status = "ยกเลิก", note = "เลื่อนไปงวด N+1"
- เพิ่มแถวงวดใหม่ที่ seq สูงกว่า

## ❓ FAQ

**Q: ทำไมต้อง 2 CSV?**
A: เพราะ metadata = 1 แปลง:1 แถว แต่ installments = 1 แปลง:N แถว — โครงสร้างต่างกัน ต้องแยก Sheet

**Q: รวมเป็น Sheet เดียวได้ไหม?**
A: เทคนิคได้ แต่ Sheet จะมีคอลัมน์เกิน 100 ตัว ลำบากในการดู เลือกแยกดีกว่า

**Q: ถ้าไม่ใช้ตารางผ่อนดาวน์ Dashboard จะพัง?**
A: ไม่ — ถ้า installments_csv_url ว่างไว้ Dashboard ก็ทำงานปกติ ไม่แสดง panel

**Q: เพิ่มงวดได้สูงสุดกี่งวดต่อแปลง?**
A: ไม่จำกัด — แต่ Dashboard render เป็นตาราง scroll ได้ พอ 20+ งวดอาจเริ่มเยอะ

**Q: ลูกค้าเห็นตารางผ่อนดาวน์ผ่าน Customer view ไหม?**
A: ไม่ — ตารางอยู่ใน .salesOnly เห็นเฉพาะ Sales view

**Q: ทีม Sales จะ flag งวดเลยกำหนดยังไง?**
A: ไม่ต้อง flag เอง — ระบบเช็ค dueDate < วันนี้ + status = "รอจ่าย" จะ flag อัตโนมัติ
